# Commentary Rules

## Commentary Purpose
Analysts add explanatory notes for significant movements or review references.

## Commentary Should Support
- one latest commentary per row for MVP
- future history/versioning
- optional reference tag association

## Commentary Must Be Linked To
- report
- row
- source code
- label (display only)
- optional reference tag

## UX Expectations
When a row is selected, the commentary editor should show:
- source code
- row label
- current/prior/variance values
- reference tag
- commentary text

## Future Extensions
- commentary templates
- AI-suggested commentary
- reviewer approval
- multi-user collaboration
